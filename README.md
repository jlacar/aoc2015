# Advent of Code 2015 - In Kotlin

## General approach

I am actually backtracking to the first AoC set of problems here. Taking inspiration from Todd Ginsberg's way of solving in Kotlin, I'm using the `Solution` interface to define the shape of a solution. The interface defines the `part1()` and `part2()` functions, with the return type specified as a generic parameter. Most of the solutions will be defined as `Solution<Int>` because most of the problems expect numbers.
